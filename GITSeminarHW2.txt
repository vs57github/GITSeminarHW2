1. Просмотрите историю коммитов в своём проекте и выберите три случайных коммита. Просмотрите изменения, которые были в них сделаны.

2. Верните эти изменения командой git revert последовательно, чтобы в итоге получилось тоже три коммита.

3. Попробуйте отменить эти три коммита:
* последний — командами git reset --soft и git restore;
* предпоследний — командой git reset --mixed и git restore;
* первый — командой git reset --hard.

Для выполнения задания по отмене коммитов и восстановлению изменений нам понадобится идентификаторы коммитов, которые нужно отменить. Предположим, что у нас есть три коммита с идентификаторами "commit1", "commit2" и "commit3". Будем выполнять операции в порядке, указанном в задании.

1. Отмена изменений в коммите "commit3":
```bash
git revert commit3
```
Команда `git revert` создаст новый коммит, который отменяет изменения, внесенные в "commit3".

2. Отмена изменений в коммите "commit2":
```bash
git revert commit2
```
Аналогично предыдущему шагу, `git revert` создаст новый коммит, отменяющий изменения в "commit2".

3. Отмена изменений в коммите "commit1":
```bash
git revert commit1
```
Также создается новый коммит, который отменяет изменения в "commit1".

Теперь мы отменили все три коммита с помощью `git revert`. Если мы хотим полностью удалить эти коммиты и вернуться к состоянию перед ними, мы можем использовать различные команды `git reset`.

4. Отмена последнего коммита (commit3) с сохранением изменений:
```bash
git reset --soft HEAD~1
```
Команда `git reset --soft` отменяет последний коммит и оставляет изменения из этого коммита в рабочей директории и индексе.

5. Восстановление изменений после отмены последнего коммита:
```bash
git restore .
```
Команда `git restore .` восстанавливает изменения в рабочей директории, которые были отменены командой `git reset --soft`.

6. Отмена предпоследнего коммита (commit2) с удалением изменений:
```bash
git reset --mixed HEAD~1
```
Команда `git reset --mixed` отменяет последний коммит и сбрасывает изменения из этого коммита в индекс, но сохраняет изменения в рабочей директории.

7. Восстановление изменений после отмены предпоследнего коммита:
```bash
git restore .
```
Команда `git restore .` восстанавливает изменения в рабочей директории, которые были отменены командой `git reset --mixed`.

8. Отмена первого коммита (commit1) с удалением изменений:
```bash
git reset --hard commit1
```
Команда `git reset --hard` полностью удаляет указанный коммит и откатывает изменения в рабочей директории и индексе к состоянию, соответствующему коммиту "commit1".
